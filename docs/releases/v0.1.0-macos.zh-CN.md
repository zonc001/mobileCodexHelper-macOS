# mobileCodexHelper-macos v0.1.0

## 发布定位

这是 `mobileCodexHelper` 的首个 macOS 适配版本发布说明。

这个版本不是简单把原始 Windows 方案搬到 Mac，而是围绕 macOS 的实际使用方式，补齐了一条可运行、可远程、可审批的使用链路：

- Mac 本地运行 Codex 控制面板
- 手机浏览器查看项目 / 会话 / 消息
- 手机继续发送消息，让电脑上的 Codex 接着执行
- 首次新设备登录要求人工批准
- 支持 Tailscale 私网远程访问

## 本次版本包含什么

### macOS 本地运行支持

新增：

- `scripts/start-mobile-codex-macos.sh`
- `scripts/stop-mobile-codex-macos.sh`
- `scripts/status-mobile-codex-macos.sh`

作用：

- 在 macOS 上启动本地 mobileCodexHelper 服务
- 停止本地服务
- 查看本地服务健康状态

运行时数据默认落在：

- `.runtime/macos/auth.db`
- `.runtime/macos/logs/`

### Tailscale 远程访问支持

新增：

- `scripts/start-mobile-codex-remote-macos.sh`
- `scripts/stop-mobile-codex-remote-macos.sh`

作用：

- 一键启动本地服务并开启 `tailscale serve`
- 一键关闭远程访问并停止本地服务
- 启动后直接输出 `Phone URL`

这让项目从“只能本地或局域网用”，扩展成了“可通过 Tailnet 在外网安全访问”。

### 新设备审批 CLI

新增：

- `scripts/device-approval-cli.py`

支持：

- 查看待审批设备
- 批准新设备
- 拒绝新设备

这让 macOS 不依赖 Windows 桌面控制器，也能保留原项目最关键的安全机制：

- 首次新设备登录必须人工批准
- 已批准设备加入 trusted devices 白名单

### Finder 双击入口

新增：

- `Start Mobile Codex Remote.command`
- `Stop Mobile Codex Remote.command`
- `Mobile Codex Status.command`
- `List Pending Device Approvals.command`

作用：

- 让 macOS 用户不必记住所有命令
- 可以直接在 Finder 中双击运行常用操作

### 文档完善

新增 / 更新：

- `README.md`
- `README.macos.zh-CN.md`

## 设计原则

这个版本依然保持原项目的核心原则：

- 单用户私有使用
- 首次新设备审批
- 手机主要负责查看与聊天控制
- 不直接鼓励把本地高权限控制面板裸暴露到公网
- 远程访问优先走 Tailscale

## 当前已验证的使用链路

这次版本已经实际验证过：

- Mac 本地 `http://127.0.0.1:3001` 可访问
- 局域网访问可用
- `tailscale serve` 远程访问可用
- 手机端首次设备审批可用
- 已批准设备在普通浏览器模式下可持久化

## 当前已知限制

- 仍然没有原生 macOS GUI 控制台
- 审批目前是 CLI 方式，不是桌面窗口方式
- Tailscale 在 iPhone 上可能与其他 VPN / 代理工具互斥
- 如果手机使用无痕模式或临时 WebView，可能会因为 `localStorage` 不持久而反复触发新设备审批

## 适合谁

这个版本最适合：

- 在 Mac 上长期运行 Codex 的个人用户
- 希望在手机上远程继续控制本地 Codex 的用户
- 希望保留私有化和设备审批边界，而不是直接公网暴露本地 agent 面板的用户
